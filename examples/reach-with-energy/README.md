# Generated reach-with-energy example

This is a compact, credential-free view of one real provider-backed run. The builder
authored a movement action, energy state, failure predicate, reach objective, and
successful solution using only the generic environment-program operations.

- `accepted-environment.json` is the exact frozen program artifact from the run.
- `acting-rollout.summary.json` projects the full acting evidence down to each chosen
  action and authoritative resulting state so the complete six-step path is readable
  at a glance.

Both files carry the same environment hash. The independent actor reached the beacon
with four energy remaining. Full runs contain observations, response attempts, and
per-effect states as described in the root README; those repetitive records are omitted
from this reviewer-oriented example only.
